## 1. Problem Statement

The instinct mode dropdown exists but is placed in the middle of a scrollable frame,
making it easy to miss. More critically, when `PetMind` auto-switches modes (e.g. from
`observe` to `assist` when an overdue task is detected), the dropdown and mode label do
not update to reflect the new state unless the player manually triggers a `refresh`.
Players have no way of knowing their pet switched modes, which makes the personality
system invisible.

## 2. Goals

- The mode dropdown and mode label are relocated to the sidebar (story 5.1) where they
  are always visible.
- The mode label updates reactively whenever `PetMind` changes mode — no manual refresh
  required.
- Manual mode selection via the dropdown is preserved.
- "Auto" option in the dropdown re-enables automatic mode switching.

## 3. Non-Goals

- Animated mode transition effects.
- Mode history log or audit trail.
- Custom user-defined modes.

## 4. Personas & User Stories

- **As a player**, I want to see my pet's current instinct mode at all times so that
  I understand why it's behaving the way it is.
- **As a player**, I want to override my pet's mode to "Assist" when I need focused
  help so that it adjusts its behavior to my current need.
- **As a player**, I want the mode label to update automatically when the pet switches
  modes so that I'm never surprised by sudden behavior changes.

## 5. Functional Requirements

- **FR-1.** The system MUST display the mode label in the sidebar, updated on every
  `refresh()` call and on every auto-mode switch.
- **FR-2.** The system MUST move the mode dropdown from the scroll area to the sidebar.
- **FR-3.** The system MUST call `update_mode_display()` from within `PetMind.set_mode`
  via a registered callback, so the label updates without waiting for `refresh`.
- **FR-4.** The system MUST preserve the "Auto" dropdown option that re-enables
  `PetMind`'s automatic mode selection.
- **FR-5.** The system SHOULD visually distinguish Auto mode from Manual mode in the
  label, e.g. `"Auto: Play"` vs `"Manual: Assist"`.

## 6. Non-Functional Requirements

- **Performance** — Mode label update completes in < 10 ms (single `StringVar.set` call).
- **Maintainability** — `PetMind` must not import `TaskWindow`; callback registered
  at construction time via a `on_mode_change` hook parameter.

## 7. Acceptance Criteria

- **AC-1.** *Given* the TaskWindow is open and the pet is in Auto:Observe mode, *when*
  an overdue task triggers `PetMind.on_overdue_task`, *then* the sidebar mode label
  updates to `"Auto: Assist"` without any user action.
- **AC-2.** *Given* the player selects "Assist" from the dropdown, *when* the selection
  is applied, *then* the label shows `"Manual: Assist"` and `PetMind.manual_override`
  is `True`.
- **AC-3.** *Given* manual override is active, *when* the player selects "Auto" from
  the dropdown, *then* `manual_override` becomes `False` and the pet resumes automatic
  mode switching.
- **AC-4.** *Given* the TaskWindow is open, *when* the sidebar is visible, *then* the
  mode dropdown and label are both present in the sidebar without scrolling.

## 10. UI / UX

Sidebar additions (below XP bar, above streak):

```
Mode: Auto: Play      ← StringVar label, updates reactively
[Auto ▾]              ← OptionMenu: Auto / Play / Assist / Observe
```

- Label color: neutral gray for Auto; accent blue for Manual.
- Dropdown width: 120 px.
- Existing `mode_frame` in scroll area is removed.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| `PetMind` callback introduces circular dependency with `TaskWindow` | M | M | Pass a plain callable (lambda) at wiring time in `main.py`; `PetMind` stores only the callable, not a reference to `TaskWindow` |
| Callback fires from Pygame thread, causing Tkinter thread-safety issues | H | H | Use `window.after(0, callback)` to schedule label update on the Tkinter main thread |

## 15. Rollout Plan

- No feature flag needed.
- Two-file PR: `TaskWindow.py` + `PetMind.py`; one-liner addition in `main.py` to
  register the callback.

## 16. Test Plan

  assert mock was called once with the new mode string.

  `manual_override == False`.
- **Manual** — open TaskWindow; trigger an overdue task; confirm mode label updates
  without clicking anything. Select Manual:Play; confirm label changes. Select Auto;
  confirm label returns to Auto prefix.
