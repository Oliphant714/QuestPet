## 1. Problem Statement

Players earn XP when they complete tasks, but the TaskWindow gives no persistent visual
feedback about the pet's current level or progress toward the next level. The only
signal is a transient messagebox. Without a visible XP bar, the progression system is
invisible and fails to motivate continued task completion.

## 2. Goals

- TaskWindow sidebar shows the pet's current level and an XP progress bar.
- Completing a task updates the bar immediately.
- Level-up events display a distinct celebration message distinct from ordinary
  task-complete feedback.
- XP and level are read from `PetCore` and persisted via story 1.1.

## 3. Non-Goals

- Animated XP bar fill transition (stretch goal for a later story).
- Growth point allocation UI (separate story 3.2).
- Achievement badges or milestone rewards.

## 4. Personas & User Stories

- **As a player**, I want to see my pet's level and XP bar in the task window so that
  I always know how close I am to leveling up.
- **As a player**, I want a special message when I level up so that the moment feels
  rewarding and distinct from ordinary task completion.
- **As a developer**, I want the XP display to read directly from `PetCore` so there
  is a single source of truth.

## 5. Functional Requirements

- **FR-1.** The system MUST display the pet's current level as `"Level {n}"` in the
  TaskWindow sidebar.
- **FR-2.** The system MUST display an XP progress bar showing `xp / xp_to_next` as
  a fraction, updated after every task completion.
- **FR-3.** The system MUST show a level-up dialog distinct from the ordinary
  task-complete dialog when `PetCore.gain_xp` returns `True`.
- **FR-4.** The system MUST call `self.refresh()` (which updates the sidebar) after
  every XP-awarding event.
- **FR-5.** The system SHOULD display the numeric XP values alongside the bar,
  e.g. `"45 / 100 XP"`.

## 6. Non-Functional Requirements

- **Performance** — Sidebar refresh completes in < 50 ms (Tkinter label + canvas
  update only).
- **Maintainability** — XP bar is a reusable `tk.Canvas` widget drawn in a dedicated
  `_draw_xp_bar` helper method.

## 7. Acceptance Criteria

- **AC-1.** *Given* the TaskWindow is open and the pet is level 2 with 45/100 XP,
  *when* the sidebar is inspected, *then* `"Level 2"` and `"45 / 100 XP"` are visible.
- **AC-2.** *Given* a task worth 60 XP is completed when the pet has 80/100 XP,
  *when* the task is marked complete, *then* the pet levels up, a level-up dialog
  appears, and the sidebar shows `"Level 3"` with the correct remaining XP.
- **AC-3.** *Given* a task is completed without triggering a level-up, *when* the
  completion dialog closes, *then* the XP bar has advanced proportionally.
- **AC-4.** *Given* the app is restarted after story 1.1 is shipped, *when* the
  TaskWindow opens, *then* the sidebar shows the persisted level and XP values.

## 8. Data Model

No new fields — reads `PetCore.level`, `PetCore.xp`, `PetCore.xp_to_next` already
defined. Story 1.1 handles persistence of these values.

## 10. UI / UX

Sidebar panel (left column of `TaskWindow`, added in this story):

```
┌──────────────────┐
│  [Pet sprite]    │
│  Level 3         │
│  ████████░░ 80%  │
│  45 / 100 XP     │
└──────────────────┘
```

- Sidebar implemented as a `tk.Frame` on the left; main task list moves to the right.
- XP bar: `tk.Canvas` 160×12 px; filled rectangle redrawn on `refresh`.
- Level-up dialog: `messagebox.showinfo` with title `"Level Up!"` and text from
  `PetVoice.get_line("level_up", mode)`.
- Empty / first-run state: sidebar shows `"Level 1"` and `"0 / 100 XP"`.
