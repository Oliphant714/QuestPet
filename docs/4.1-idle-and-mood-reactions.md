## 1. Problem Statement

`PetMind.idle_events` increments on every idle tick but never resets, so after two idle
cycles the pet locks into `observe` mode permanently regardless of task completions.
This makes the instinct system feel broken — completing tasks after a period of
inactivity has no effect on the pet's mode or demeanor. Players perceive the pet as
unresponsive and stop engaging with it.

## 2. Goals

- `idle_events` resets to 0 when the player completes a task or interacts with the UI.
- Idle notifications respect a cooldown so the pet doesn't spam the status label.
- The pet transitions to the sleeping animation after a configurable extended idle
  period (default 5 minutes).
- Mode correctly reflects the most recent meaningful event, not accumulation of idle ticks.

## 3. Non-Goals

- Emotional decay curves or weighted mood history.
- User-configurable idle thresholds via a settings screen.
- Sounds or audio cues on idle transitions.

## 4. Personas & User Stories

- **As a player**, I want my pet to notice when I complete tasks and return to an
  engaged state after being idle so that it feels like a responsive companion.
- **As a player**, I want my pet to go to sleep if I leave it alone for a long time
  so that its behavior feels natural and alive.
- **As a developer**, I want idle event counters to reset on meaningful activity so
  that the mood FSM doesn't drift into a permanent state.

## 5. Functional Requirements

- **FR-1.** The system MUST reset `PetMind.idle_events` to 0 whenever
  `on_task_completed` is called.
- **FR-2.** The system MUST reset `PetRenderer.last_user_activity` timestamp when the
  player interacts with the TaskWindow (add, edit, delete, complete).
- **FR-3.** The system MUST transition the pet sprite to `"idle_to_sleeping"` if the
  user has been inactive for >= 5 minutes (300 000 ms).
- **FR-4.** The system MUST NOT fire another idle notification within `idle_repeat_ms`
  (currently 15 s) of the last one.
- **FR-5.** The system SHOULD resume `"sleeping_to_idle"` animation automatically when
  the user next interacts after the pet has slept.

## 6. Non-Functional Requirements

- **Performance** — `check_idle_state` runs every frame (5 fps); must complete in < 1 ms.
- **Reliability** — Timer comparisons use `pygame.time.get_ticks()` consistently;
  no mixing with `datetime.now()`.

## 7. Acceptance Criteria

- **AC-1.** *Given* the pet has been idle for 2 cycles (mode = observe), *when* the
  player completes a task, *then* `idle_events` is 0 and mode shifts to `play`.
- **AC-2.** *Given* the pet is idle, *when* 5 minutes pass without any interaction,
  *then* the pet sprite transitions to the sleeping animation.
- **AC-3.** *Given* the pet is sleeping, *when* the player presses a key or opens the
  task window, *then* the pet plays `sleeping_to_idle` and returns to idle.
- **AC-4.** *Given* an idle notification was just shown, *when* another 14 seconds pass,
  *then* no second notification fires; it fires only after the full `idle_repeat_ms`.
