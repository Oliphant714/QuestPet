## 1. Problem Statement

The current `TaskWindow` is a single vertical column of widgets with no visual hierarchy.
Pet stats (level, XP, streak) are not visible at all in the window — players must infer
progression entirely from voice-line popups. This disconnects the task list from the
pet companion, making QuestPet feel like two separate tools rather than one integrated
experience.

## 2. Goals

- TaskWindow is split into a fixed left sidebar (pet stats) and a scrollable right
  panel (task list).
- Sidebar shows: pet name, level, XP bar, streak count, and current instinct mode.
- Task list panel retains all existing functionality (add, edit, delete, complete).
- Layout is stable at the default 600×400 window size.

## 3. Non-Goals

- Resizable panes or drag-to-resize sidebar.
- Pet sprite rendered inside the Tkinter window (pet lives in the Pygame overlay).
- Dark mode or theme switching.

## 4. Personas & User Stories

- **As a player**, I want to see my pet's level and streak without leaving the task
  list so that I stay motivated while managing my quests.
- **As a player**, I want the task list and pet stats in the same window so that
  QuestPet feels like a unified companion app, not a plain to-do list.

## 5. Functional Requirements

- **FR-1.** The system MUST render a left sidebar Frame of fixed width 180 px containing
  level label, XP bar, XP numeric label, and streak label.
- **FR-2.** The system MUST render a right panel containing the existing task listboxes,
  action buttons, and mode dropdown.
- **FR-3.** The system MUST update sidebar values on every call to `self.refresh()`.
- **FR-4.** The system MUST set the window geometry to at least `600x400`.
- **FR-5.** The system SHOULD display the current instinct mode in the sidebar as
  `"Mode: Auto: Observe"` using the string from `event_router.get_mode_display()`.

## 6. Non-Functional Requirements

- **Performance** — Full refresh (sidebar + listboxes) completes in < 100 ms.
- **Maintainability** — Sidebar built in a `_build_sidebar(parent)` method; right panel
  built in `_build_task_panel(parent)`. `create_window` calls both.

## 7. Acceptance Criteria

- **AC-1.** *Given* the TaskWindow is opened, *when* the window renders, *then* a
  sidebar is visible on the left with level and XP information.
- **AC-2.** *Given* a task is completed and XP is awarded, *when* `refresh` is called,
  *then* the sidebar XP bar and numeric label update to reflect the new values.
- **AC-3.** *Given* the window is resized to 600×400, *when* the layout is inspected,
  *then* neither panel clips or overlaps the other.
- **AC-4.** *Given* the event router is connected, *when* the sidebar is viewed,
  *then* the mode label shows the current mode string (e.g. `"Auto: Observe"`).

- Sidebar background: light gray (`#f5f5f5`).
- XP bar: `tk.Canvas` 140×10 px, filled rectangle.
- Mode label updates on every `refresh`.
- Right panel retains `ScrollableFrame` wrapper.

## 14. Risks & Mitigations

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Tkinter `grid` vs `pack` conflicts when combining sidebar + scrollable frame | M | M | Use `pack` with `side=LEFT` for sidebar and `side=RIGHT/fill=BOTH/expand=True` for panel |
| Sidebar too narrow for long mode strings | L | L | Set `wraplength=160` on mode label |
