## 1. Problem Statement

Deleting a task removes it without confirmation, making accidental deletion unrecoverable.
The complete flow works but does not gracefully handle the case where the selected task
has already been marked complete (e.g. via a race condition), leaving the user with no
feedback. These gaps erode trust in the task list as a reliable productivity tool.

## 2. Goals

- Deleting a task requires a confirmation dialog before removal.
- Completing a task that is already complete shows a clear message rather than silently
  doing nothing.
- Both actions save state and refresh the listbox on success.
- No selection produces a consistent, descriptive warning.

## 3. Non-Goals

- Undo / redo for deletions.
- Batch delete or batch complete.
- Moving completed tasks back to active.

## 4. Personas & User Stories

- **As a player**, I want to confirm before deleting a task so that I don't lose a
  quest by accidentally clicking Delete.
- **As a player**, I want to mark a quest as complete and see my pet react immediately
  so that the reward loop feels satisfying.
- **As a player**, I want a clear message if I try to complete a task that is already
  done so that I understand why nothing happened.

## 5. Functional Requirements

- **FR-1.** The system MUST show a `messagebox.askyesno` confirmation before removing a
  task from `active_tasks`.
- **FR-2.** The system MUST call `task_manager.save_state()` and `self.refresh()` only
  if the user confirms deletion.
- **FR-3.** The system MUST show a `messagebox.showwarning` if the user clicks Complete
  but the selected task is already marked `completed == True`.
- **FR-4.** The system MUST show a `messagebox.showwarning` if Complete or Delete is
  clicked with nothing selected in the listbox.
- **FR-5.** The system SHOULD show the XP awarded in the completion info dialog (already
  partially implemented via `PetEventRouter.on_task_completed`).

## 6. Non-Functional Requirements

- **Performance** — Confirmation dialog appears in < 100 ms.
- **Maintainability** — Delete and Complete remain separate methods in `TaskWindow`; no
  shared state between them.

## 7. Acceptance Criteria

- **AC-1.** *Given* a task is selected, *when* Delete is clicked and the user clicks
  "No" in the confirmation dialog, *then* the task remains in the list and the save
  file is unchanged.
- **AC-2.** *Given* a task is selected, *when* Delete is clicked and the user confirms,
  *then* the task is removed from the listbox and absent from the next save file.
- **AC-3.** *Given* no task is selected, *when* Complete is clicked, *then* a warning
  dialog `"Please select a task to complete."` appears.
- **AC-4.** *Given* a task is selected and completed successfully, *when* Complete is
  clicked, *then* the task moves to the completed listbox and the pet plays its
  reaction animation.
- **AC-5.** *Given* a task that is already `completed`, *when* Complete is clicked,
  *then* a warning `"This task is already complete."` appears and no XP is awarded.

## 10. UI / UX

- Delete confirmation: `"Delete '{task.title}'? This cannot be undone."` with Yes/No.
- Complete success dialog: driven by `PetEventRouter` response string (already
  implemented); shows pet's voice line + XP awarded.
- Already-complete warning: `"This task is already complete."` via `showwarning`.
- No-selection warning: `"Please select a task to complete."` / `"…to delete."`.
